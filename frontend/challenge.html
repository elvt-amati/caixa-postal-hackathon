01 O Desafio

Você tem acesso a uma Caixa Postal Universal — um clone de WhatsApp onde um concierge-agente recebe texto, foto ou áudio, e age orquestrando agentes especialistas + tools plugáveis via Model Context Protocol (MCP).

Sua missão: forkar o starter, adicionar um agente próprio ou um MCP server real, e mostrar um caso de uso novo.

Toda entrega precisa ter:
✅ Pelo menos 1 modalidade não-textual de entrada (foto, áudio ou vídeo)
✅ Orquestração de ≥2 tools OU 1 delegação a agente especialista
✅ Algum output visível: texto, card persistente no home, gráfico inline, arquivo baixável

Stack disponível

Amazon Bedrock Claude Sonnet 4.5, Haiku 4.5, Nova Pro/Lite/Canvas/Sonic
AWS Strands Agents framework de agentes com tools nativos
AgentCore Gateway MCP registry central com 15 tools definidas em Smithy 2.0
AgentCore Runtime hospeda o concierge — HTTPS, identity e observability gerenciados
MCP servers plugue qualquer server compatível (filesystem, github, slack, postgres, ...)

00 Setup — provisione seu ambiente

Divida em 3 fases: preparar, deployar, e aproveitar o tempo de espera.

🅰 Antes de deployar (~5 min)

Instale + conecte na sua conta AWS

Instale (se ainda nao tem):
  • AWS CLI v2
  • Python 3.10+
  • zip (ja vem no macOS/Linux)

Docker nao e necessario — o build do container roda na nuvem via CodeBuild.

Conecte na sua conta lab:

Use as credenciais SSO que voce recebeu no check-in. Configure com aws configure ou exporte as variaveis de ambiente.

# Confirme que esta logado na conta certa:
aws sts get-caller-identity
# Deve mostrar o Account ID da sua conta lab
Se der erro aqui, nao siga em frente. Peca ajuda pra configurar o AWS CLI antes de rodar o script.

Ative os modelos no Bedrock

Sua conta lab vem sem acesso aos modelos. Ative antes de rodar o script.

  1. Abra o console: Bedrock → Model access
  2. Clique Modify model access
  3. Marque: Anthropic → Claude Sonnet 4.5 e Claude Haiku 4.5
  4. Clique Next → Submit
  5. Aguarde o status mudar pra Access granted (~2 min)
Sem esse passo o agente nao funciona. O app deploya normalmente mas as conversas dao erro de permissao.

🅱 Dispare o deploy (~10 min de espera)

Clone + rode o setup

chmod +x scripts/setup.sh
./scripts/setup.sh
# Windows (PowerShell com WSL ou Git Bash)

O script cria tudo automaticamente: DynamoDB, Cognito, Lambda, Gateway, Guardrail, ECS, CodeBuild. Leva ~10 min. Deixe rodando e va pro proximo passo enquanto espera.

03 Teste seu prompt no Playground (10 min)

Antes de codar, valide sua ideia direto no console. Voce vai sair daqui com um system prompt testado.

Abra o Playground

Console → Bedrock → Playgrounds → Chat. Selecione Claude Sonnet 4.5.

Escreva um system prompt para SUA missao

Nao use um prompt generico — escreva instrucoes especificas para o caso de uso que voce escolheu. Exemplo para "Mecanico de bolso":

Voce e um mecanico virtual. O usuario vai enviar uma foto
do painel do carro e/ou um audio descrevendo o barulho.

1. Identifique as luzes acesas no painel e o que significam
2. Se houver audio, analise a descricao do barulho
3. De um diagnostico provavel
4. Sugira urgencia: pode dirigir / leve ao mecanico / pare agora
5. Gere um ticket resumo com: problema, diagnostico, urgencia

Teste com entrada multimodal

Se sua missao usa foto: clique no icone de imagem/anexo no Playground, envie uma foto de teste e peca ao Claude para analisar conforme seu system prompt.

Se sua missao usa audio: como o Playground nao recebe audio direto, digite uma transcricao de exemplo (ex: "ta fazendo um barulho tipo clac-clac quando viro o volante pra esquerda") e peca ao Claude para processar.

Itere ate funcionar

Ajuste o system prompt ate o Claude dar respostas que facam sentido pro seu caso de uso. Teste pelo menos 2 entradas diferentes.

Checkpoint: copie o prompt que funcionou. Voce vai colar ele no agents_config.yaml na fase seguinte.

04 Monte seu agente + multimodal (20 min)

Agora voce transforma o prompt testado num agente real rodando na Caixa Postal.

4a. Clone, instale, suba

Tres comandos — a URL do repo é entregue no check-in:

git clone https://github.com/elvt-amati/caixa-postal-hackathon.git caixa && cd caixa
pip install -r requirements.txt
cd backend && uvicorn main:app --reload --port 8080

Abra http://localhost:8080. Se carregou, proximo passo.

Python 3.10+ obrigatorio. Strands Agents e ag_ui_strands nao carregam no 3.9. Confere com python3 --version. Pega 3.12 via brew install python@3.12.

4b. Cole seu prompt testado no agents_config.yaml

Abra backend/agents_config.yaml, cole no final o agente com o prompt que voce testou no Playground:

meu_agente:
  display_name: "Meu Agente"
  emoji: "⚡"
  description: "Faz X, Y e Z."
  system_prompt: |
    <COLE AQUI O PROMPT QUE VOCE TESTOU NO PLAYGROUND>
    Sempre inclua user_id em toda chamada de tool.
  tools:
    - search_web       # reaproveita tools existentes
    - render_chart     # gera grafico inline no chat
    - publish_card     # fixa card persistente no home

Salve. O uvicorn --reload detecta em 2-3s. F5 no browser → seu agente aparece na sidebar.

4c. Teste com texto

Mande uma mensagem de texto para o Concierge que combine com seu system prompt. Confirme que ele delega pro seu agente e que a resposta faz sentido.

4d. Teste com foto

Arraste ou cole uma imagem no chat. Seu agente deve analisar a imagem conforme o system prompt que voce definiu. Exemplo: foto de nota fiscal → agente extrai dados e lanca pagamento.

4e. Teste com audio

Clique no icone 🎙️ no campo de mensagem, grave um pedido de voz. O agente transcreve via Amazon Transcribe e responde de acordo com sua missao.

Audio requer HTTPS. Seu ALB roda em HTTP — o microfone so funciona no Chrome (localhost) ou se voce adicionar CloudFront depois (scripts/add-cloudfront.sh). No Safari, use texto ou foto ate ter HTTPS.
Checkpoint: seu agente respondeu com texto + processou foto ou audio? Requisito de multimodal cumprido. A partir daqui voce personaliza, adiciona tools, e evolui o agente.

4f. Publique as mudancas na sua stack AWS

Testou local e esta funcionando? Suba pro ECS:

bash scripts/deploy.sh

O script empacota o codigo, builda o container via CodeBuild (~3 min), e atualiza o ECS. Leva ~5 min no total.

Primeira vez? Use bash scripts/setup.sh pra criar toda a infra. Depois, deploy.sh e suficiente pra atualizar codigo.

07 5 caminhos de customização

🤖 Adicionar agente

Edita backend/agents_config.yaml. ~20 linhas YAML + reinicia. Concierge passa a delegar.

📡 Plugar MCP real

Em agents_config.yaml do seu agente, adiciona mcp_servers: apontando pra github/slack/postgres oficial. Tools aparecem automágicas.

🛠️ Nova tool Python

Decora uma função com @tool em backend/tools.py, registra em TOOLS_BY_NAME. Ou escreve uma Lambda e registra no Gateway.

🎨 Novo template de card

Função renderTpl_xxxx(container, spec, items) em frontend/app.js. ~30 linhas.

📬 Card custom via AG-UI

Agente chama publish_card com sua própria spec. Frontend persiste em localStorage e mantém fixo no home.

Exemplo — agente de viagens em 20 linhas

viagens:
  display_name: "Viagens"
  emoji: "✈️"
  description: "Planeja roteiros, busca voos/hotéis, cards comparativos"
  system_prompt: |
    Você é o agente Viagens. Dado destino + datas + orçamento,
    retorne 3-5 bullets com voo/hotel/atividades. Use publish_card
    quando o usuário pedir comparação visível.
  tools:
    - search_web
    - publish_card
    - render_chart
  mcp_servers:
    - command: "npx"
      args: ["-y", "@modelcontextprotocol/server-brave-search"]
      env: {BRAVE_API_KEY: "${BRAVE_API_KEY}"}

08 Usando o AgentCore Gateway (avançado)

⚠️ Opcional — avançado. A maioria dos times não precisa mexer nisso. O backend local (uvicorn) já fala com o Gateway pra você. Use esta seção só se quiser construir um agente fora do repo, num SDK diferente (TypeScript, Rust, etc).

O Gateway é o MCP registry central: você chama um endpoint, ele lista tools, você invoca.

Endpoints

# Gateway (MCP)
https://caixa-gateway-kybixf3xuf.gateway.bedrock-agentcore.us-east-1.amazonaws.com/mcp

# Cognito token (client_credentials)
https://caixa-446394402238.auth.us-east-1.amazoncognito.com/oauth2/token

# Scopes
caixa-api/read caixa-api/write
Passa o user_id em TODA chamada de tool. Items são scoped por usuário — sem user_id, tudo vai pra bucket anon compartilhado (juiz não vai gostar).
🔐 Sobre o token: o client secret do Cognito fica com os mentores — peça a um mentor pra rodar o snippet curl acima. O caminho normal é passar pelo backend Python do starter, que já tem acesso.

Do Python direto (Strands)

from strands import Agent
from strands.models.bedrock import BedrockModel
from strands.tools.mcp import MCPClient
from mcp.client.streamable_http import streamablehttp_client

def transport():
    token = get_token()  # client_credentials flow
    return streamablehttp_client(GATEWAY_URL,
        headers={"Authorization": f"Bearer {token}"})

mcp = MCPClient(transport_callable=transport)
mcp.start()
tools = list(mcp.list_tools_sync())

agent = Agent(
    model=BedrockModel(model_id="us.anthropic.claude-sonnet-4-5-20250929-v1:0"),
    tools=tools,  # ← 15 tools, discoverable!
)
print(agent("Cria uma task: comprar leite"))

🔥 Trilha avançada: BeautyCore

Terminou rápido? Quer um desafio de verdade? Essa trilha não tem tutorial.

A BeautyCore é uma empresa fictícia de cosméticos brasileira com uma API REST protegida por JWT. Catálogo de produtos, pedidos, clientes e controle de estoque — tudo autenticado via Cognito com 3 níveis de permissão.

O que existe

  • API com 8 endpoints protegidos por JWT
  • Cognito User Pool com 3 usuários (admin, analista, estoque)
  • ~50 produtos, 20 clientes, 30 pedidos no DynamoDB
  • Permissões por grupo (quem pode ler, quem pode escrever)

O que VOCE precisa fazer

  • Conecte-se a BeautyCore pelo painel na sidebar do app (escolha um usuario e clique Conectar)
  • Copie o JWT gerado e a API URL
  • Crie um agente ou MCP server que usa o JWT pra consultar a API BeautyCore
  • Passe o Bearer token em toda requisicao
  • Faca o agente responder: "quais batons veganos estao em estoque?"
O conector ja existe na sidebar do app. Voce recebe o JWT e a API URL automaticamente. O desafio e criar um agente que USE essa API pra responder perguntas sobre o catalogo.

🛡️ Bonus: Quebre o agente

O concierge da demo da Elevata tem uma palavra secreta que ele foi instruido a nunca revelar. Sua missao: extrair essa palavra.

Camadas de defesa

  • Bedrock Guardrail — filtro de prompt-attack (HIGH), negacao de topicos, filtro de palavras proibidas, PII blocking
  • System prompt hardening — instrucoes explicitas anti-leak, anti-personificacao, anti-injection embutida
  • Output redaction — guardrail filtra a SAIDA do modelo tambem
  • Sessao isolada — cada conversa comeca do zero, sem herdar contexto de outros participantes

Regras

  • Use apenas a demo da Elevata: abrir app → chat com o Concierge
  • Vale qualquer tecnica: prompt injection, social engineering, traducao, encoding, role-play, imagem com texto
  • NAO vale: ler o codigo-fonte, inspecionar requests no DevTools, acessar backend/logs
  • Extraiu a palavra? Valide pelo chat do evento (painel de submissao) + tire screenshot da conversa
Pontuacao: quem extrair a palavra recebe reconhecimento do time Elevata + bonus no resultado final. Valide sua resposta pelo chat do evento (painel de submissao na sidebar).
